# Глава 8 · Самоэволюция агента

> Рост без изменения весов. Три парадигмы обучения, обучение на опыте и путь от «пользователя инструментов» к «создателю инструментов» — как агенту перейти от «умного» к «умелому».

## Сопутствующие проекты

| Проект | Тип | Описание |
| --- | :--: | --- |
| [gaia-experience](gaia-experience/) | ✅ | На базе фреймворка AWorld и бенчмарка GAIA реализует полный цикл «обучение — применение». Агент автоматически суммирует успешные траектории задач в структурированный опыт, извлекает и применяет его в новых задачах, достигая самоэволюции. |
| [browser-use-rpa](browser-use-rpa/) | ✅ | Реализует систему записи рабочих процессов для автоматизации браузера, автоматически инкапсулируя повторяющиеся последовательности операций в параметризованные инструменты. Переходя от дорогого вывода LLM к точному автоматическому исполнению, достигает ускорения в 3–5 раз. |
| [prompt-distillation](prompt-distillation/) | ✅ | Дистиллирует эффективность сложных промптов в параметры модели, сокращая длину промпта на инференсе и закрепляя контекстный опыт в параметризованном знании. |
| [prompt-auto-optimization](prompt-auto-optimization/) | ✅ | Автоматическое обучение системного промпта на основе обратной связи человека: на примере проблемы «избыточной передачи» в авиа-поддержке в стиле tau-bench кодинг-агент читает файл системного промпта, выявляет проблемные правила, генерирует точные правки и реально переписывает файл промпта. Затем повторно оценивает изменения, образуя цикл «обратная связь → переписывание → проверка». |
| [self-evolving-tools](self-evolving-tools/) | ✅ | Подход в духе Alita «минимум предопределённого, максимум самоэволюции»: у агента нет заранее готовых доменных инструментов, лишь пять общих мета-инструментов. Столкнувшись с невыполнимой задачей, он ищет в сети открытые библиотеки/API, читает документацию, тестирует их в песочнице, инкапсулирует рабочие решения как новые инструменты, сохраняет их в библиотеке для повторного использования и на всём пути делает упор на контроль галлюцинаций. |
| [self-evolution-eval](self-evolution-eval/) | ✅ | Специальный набор данных и методология проверки для оценки способности агента к «самоэволюции» (самостоятельно обнаруживать, создавать и переиспользовать инструменты): 20 кросс-доменных задач (без подсказки имён инструментов) + четырёхслойный иерархический harness проверки + управляемый эталонный агент. Выходит за рамки проверки «верен ли результат», оценивая качество обнаружения, создания и повторного использования. |
