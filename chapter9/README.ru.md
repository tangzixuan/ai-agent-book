# Глава 9 · Мультимодальность и взаимодействие в реальном времени

> Расширяет восприятие и действие с текста на голос, GUI и физический мир. Три голосовые парадигмы (каскадная / сквозная полномодальная / полнодуплексная), потоковое восприятие и синтез речи, Computer Use и манипуляции роботов.

## Сопутствующие проекты

| Проект | Тип | Описание |
| --- | :--: | --- |
| [live-audio](live-audio/) | ✅ | Демо голосового чата в реальном времени, объединяющее распознавание речи, диалог с ИИ и синтез речи. Поддерживает несколько провайдеров ИИ-сервисов (OpenAI, OpenRouter, ARK, Siliconflow), обеспечивая диалог с низкой задержкой. |
| `browser-use/` | 📖 | Browser-Use — мощный фреймворк автоматизации браузера, позволяющий LLM управлять браузером для выполнения сложных задач. Поддерживает заполнение форм, навигацию по сайтам и извлечение данных — типичная реализация автоматизации GUI (Computer Use). |
| `claude-quickstarts/` | 📖 | Примеры для быстрого старта и лучшие практики работы с Claude API, охватывающие разные сценарии использования. |
| [phone-agent](phone-agent/) | ✅ | Демонстрирует голосового агента, «взаимодействующего с внешним миром по телефону от имени пользователя»: верхний слой — обычный ReAct-агент. Получив задачу на естественном языке, он самостоятельно определяет номер и цель звонка, вызывает инструмент `make_phone_call` (на абстракции телефонного API), проводит весь разговор, читает структурированный лог звонка, при необходимости перезванивает с уточнениями и в итоге отчитывается пользователю. |
| [end-to-end-speech](end-to-end-speech/) | ✅ | Соответствует сквозному речевому мышлению Step-Audio R1 («слушать → думать → говорить»); сравнивает задержку и потерю паралингвистики с каскадом ASR→LLM→TTS. |
| [streaming-speech](streaming-speech/) | ✅ | Демонстрирует ключевой компромисс потокового восприятия речи: непрерывное аудио режется на сегменты растущей длины и подаётся в ASR. Каждый полученный сегмент даёт «текущий частичный результат распознавания», обеспечивая крайне низкую задержку первого фрагмента для раннего вывода текста. Плата за это — ранние фрагменты без контекста второй половины фразы могут быть ошибочны, постепенно сходясь по мере накопления аудио. Это контрастирует с высокоточным, но высоколатентным подходом «дождаться всей фразы перед распознаванием». |
| [controllable-tts](controllable-tts/) | ✅ | Вывод основного LLM несёт управляющие токены (эмоция/темп/стиль/пауза/смех). Слой исполнения парсит эти токены, отображает их на соответствующие стилевые профили в библиотеке эталонной речи и затем синтезирует речь. Так решения «где сделать паузу и каким тоном» делегируются LLM, и один и тот же текст можно синтезировать в разных стилях и с разными эмоциями. |
