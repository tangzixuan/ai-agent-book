# Глава 10 · Многоагентное взаимодействие

> Коллективный интеллект может превзойти индивидуальный. Классификация мультиагентных систем, когда они действительно превосходят одиночного агента, сотрудничество с общим контекстом и без него, режимы сбоев и эмерджентное «общество агентов».

## Сопутствующие проекты

| Проект | Тип | Описание |
| --- | :--: | --- |
| `use-computer-while-calling/` | 📖 | Реализует архитектуру сотрудничества двух агентов — агента телефонного звонка и агента Computer Use. Агенты общаются напрямую через WebSocket без координатора. Телефонный агент ведёт голосовое взаимодействие, а компьютерный выполняет автоматизацию браузера, работая параллельно над сложными задачами, требующими и голоса, и веб-операций. |
| [staged-system-prompt](staged-system-prompt/) | ✅ | Один и тот же кодинг-агент на разных этапах исполнения задачи (уточнение требований → реализация кода → ревью кода) загружает разные системные промпты и наборы инструментов. Так он играет разные роли и ведёт себя по-разному в рамках одного диалога, при этом история диалога и состояние задачи непрерывно разделяются между этапами. Если ревью не пройдено, можно откатиться к этапу реализации. |
| [multi-role-transfer](multi-role-transfer/) | ✅ | Демонстрирует цепочечную передачу под общим контекстом: в одной сессии несколько специализированных ролевых агентов, у каждого свой системный промпт и выделенный набор инструментов. С помощью инструмента `transfer_to_agent` агент сам решает, когда переключиться на другую роль по ходу задачи. Поскольку они разделяют одну историю диалога, полный контекст естественно сохраняется при передаче. |
| [book-translation](book-translation/) | ✅ | По паттерну оркестратора раскладывает перевод длинного документа на специализированных агентов глоссария/перевода/вычитки. Менеджер хранит лишь задачи, планы, записи вызовов и индексы файлов; полный переведённый текст пишется на диск, удерживая контекст примерно постоянным. Сравнивает это с одноагентным подходом, на реальных счётчиках токенов показывая, как контролировать взрывной рост контекста и обеспечивать единообразие по всей книге через общий глоссарий. |
| [parallel-web-research](parallel-web-research/) | ✅ | Демонстрирует параллельный поиск несколькими однородными агентами с центральной координацией: главный координатор одновременно запускает N субагентов, каждый обращается к одному источнику в поисках ответа. Как только один достигает цели, остальные корректно останавливаются. Шина сообщений, параллельная диспетчеризация, мониторинг в реальном времени, каскадное завершение и обработка гонок реализованы по-настоящему (с управляемыми имитируемыми источниками вместо реального браузера). |
| [voice-werewolf](voice-werewolf/) | ✅ | На мультиагентной игре «Мафия» демонстрирует контроль доступа к информации в условиях «неразделяемого контекста»: каждый игрок — независимый LLM-агент со строго изолированным приватным контекстом. Детерминированный судья на коде решает, какая информация попадёт в контекст какого игрока, регистрирует её для аудита и в конце игры автоматически проверяет корректность изоляции. Голос — опциональное усиление. |
