# 实验 3-13：从结构化数据中提取隐性知识——以司法判例分析为例

配套《深入理解 AI Agent》第 3 章。演示如何让 Agent 不把知识库当成"只能检索的静态仓库"，
而是**先把数据读懂、提炼出结构化的决策逻辑，再基于这套逻辑回答问题**。

以「盗窃罪」判例为例，完整走通三段流水线：

```
判例文本 ──①LLM结构化抽取──▶ 结构化因子 ──②回归/重要性建模──▶ 判决经验模型
                                                                    │
                            新案情 ──③对话Agent（引用因子与权重给出量刑建议）◀──┘
```

## 三段流水线

**① 知识提取（`extractor.py`）**
用 LLM（`gpt-4o-mini`，`response_format=json_object`）从判决书事实段落中抽取结构化因子：
盗窃金额、是否前科/累犯、入户盗窃、携带凶器、团伙作案、自首、退赃退赔、认罪认罚。
文本未提及的因子返回 `null`（供第 ③ 步判断"还缺什么信息"）。抽取结果缓存到
`data/extracted.jsonl`，避免重复调用花钱；并与数据集里的 `gold` 真值对比，报告抽取准确率。

**② 因子重要性建模（`factor_model.py`）**
把因子翻译成数值特征（金额取 `ln`，是非情节取 0/1），拟合两套等价线性回归：
- **标准化模型**：系数经标准化后可横向比较 → 得到**因子重要性排序**；
- **原始尺度模型**：系数就是"该情节使刑期增/减多少个月" → 用于对单个案件做**逐因子刑期拆解**。

产出一个可解释的结构化「判决经验模型」`data/factor_model.json`（含各因子权重、拟合优度）。

**③ 对话式量刑建议 Agent（`advisor_agent.py`）**
把「判决经验模型」当决策逻辑用：从用户口语描述抽取已知因子 → 对照重要性排序，
**按重要性顺序追问仍缺失的关键因子** → 信息补全后用模型预测刑期，并把预测拆解为各因子
的贡献月数 → 让 LLM 把这些**真实数字**组织成一段引用关键因子及其权重的建议（附免责声明）。
所有刑期数字均来自模型，LLM 只负责"把数字讲清楚"，不自行编造。

## 运行

```bash
pip install -r requirements.txt
cp env.example .env        # 填入 OPENAI_API_KEY
python generate_data.py    # 可选：重新生成合成判例数据集（已自带 data/cases.jsonl）
python demo.py             # 跑通 抽取 → 建模 → 对话建议 全流程
```

首次运行会对约 48 条案例逐条调用 LLM 抽取（结果写入缓存，再次运行直接命中缓存，几乎免费）。

## 真实运行输出（节选）

```
各字段抽取准确率(对比 gold): amount 100% / prior_record 100% / restitution 100% / confession 100% ...

因子重要性排序(标准化权重; 原始系数=月/因子):   训练集 R²=0.993  5 折交叉验证 R²=0.989
   1. 盗窃金额(对数)  权重=+6.32 (+5.3月/ln单位)  从重
   2. 前科/累犯      权重=+5.49 (+11.1月/情节)   从重
   3. 自首         权重=-4.49 (-9.1月/情节)    从轻
   4. 入户盗窃      权重=+3.56 (+7.2月/情节)    从重
   5. 退赃退赔      权重=-3.25 (-6.6月/情节)    从轻
   ...

对话 Agent：识别到金额缺失 → 追问金额/退赃/认罪认罚 → 补全后预测约 53.7 个月，
并指出「前科+11.1 个月、入户+7.2 个月」为主要从重因子。
```

学到的原始系数与合成数据的"真值量刑公式"高度吻合（前科 11.1↔11、自首 -9.1↔-9、
入户 7.2↔7……），说明模型确实从数据中把隐性的判决规律"学"了回来——这就是可验证的真实结论。

## 数据说明

`data/cases.jsonl` 是**自带的小样本合成数据**（48 条），由 `generate_data.py` 用一个"已知"的
量刑公式加噪声生成：每条含自然语言 `fact`、结构化真值 `gold`、刑期 `label_months`。
之所以合成，是因为 CAIL2018 原始数据集体量太大、不便随仓库分发；合成数据的好处是"真值公式"
已知，可以直接验证第 ② 步是否真的把规律学了回来。

**换成真实 CAIL2018**：把 `generate_data.py` 换成读取 CAIL 的 `data_*.json`
（每行含 `fact` 与 `meta.term_of_imprisonment`），产出同样的 `data/cases.jsonl`
（每行至少含 `fact` 与 `label_months`）即可，抽取/建模/对话三段代码无需改动。
若要迁移到其他罪名，只需在 `schema.py` 里扩展因子列表。

## 文件

| 文件 | 作用 |
|------|------|
| `schema.py` | 判决因子模式定义（数值 + 是非情节） |
| `generate_data.py` | 合成小样本判例数据集 |
| `extractor.py` | 阶段 ①：LLM 结构化抽取（带缓存、准确率评估） |
| `factor_model.py` | 阶段 ②：因子重要性建模（回归 + 拆解） |
| `advisor_agent.py` | 阶段 ③：对话式量刑建议 Agent |
| `demo.py` | 全流程演示入口 |
| `config.py` | OpenAI 客户端与模型配置 |
